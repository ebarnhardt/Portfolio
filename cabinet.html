Cabinet

web application for saving and sorting online content

Idea generation, UX documentation, user research, wireframing, UI design, promotional site prototype, user testing

The Challenge

Save, Sort & Share Everything

With so many products out there for saving content - Evernote, Pinterest, Dragdis, Pocket- it can be difficult to know which one to choose. Some do a good job saving just images or just articles, but none of them do an excellent job saving a variety of content in a visually appealing way. Cabinet allows you to save a variety of content - websites, images and text documents all in one place. You can file this content into different project folders that you can then share with collaborators. It is the perfect solution for visual users to organize everything they find on the internet.
Research + Discovery

Content Keepers

I started this project with some research into online habits. I posted a survey that asked participants about the various ways that they save and create content online. Using the data from the survey, I made a plan for how the app should be structured as well as a list of features that users would like.

User Flows

To explore how users would navigate around the site, I created flows for different scenarios including when new users visit the homepage or sign up, when returning users sign in, and when all users access the dashboard. These exercises are helpful in figuring out faster ways for users to accomplish their goals. Below is Terrence's flow to send an image to a teammate.

User Testing

I ran a variety of tests on both the dashboard and the promotion page using Invision, UsabilityHub and Peek by UserTesting. I received some positive feedback from testers including that they found the site trustworthy as well as valuable. One test looked at preference for the location of the add button. Since users preferred it be placed with the left side navigation, I moved it there since it is one of the most important features of the dashboard.
